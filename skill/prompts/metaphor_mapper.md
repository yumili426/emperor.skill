# 隐喻映射器

## 任务

将用户输入的现实事件，映射为宫廷事件。

## 文件优先级

当同一映射在多个文件中出现时，按以下优先级使用：
1. `events/mappings/work_scene.md` — 工作场景（最高优先级）
2. `events/mappings/social_scene.md` — 社交场景
3. `events/mappings/emotional_scene.md` — 情感场景
4. 本文件 — 通用回退（最低优先级）

本文件提供通用映射表作为回退。场景特定的映射规则参见对应 mapping 文件。

## 映射规则

### 核心原则
- 保持事件的本质不变
- 用宫廷语言重新包装
- 保留情感色彩
- 让用户感受到"掌控感"

### 映射流程

```
[Step 1] 识别现实事件类型
  - 工作相关 → 朝堂事件
  - 情感相关 → 后宫事件
  - 社交相关 → 宴会事件
  - 负面事件 → 危机事件

[Step 2] 选择映射模板
  - 根据事件类型选择
  - 选择合适的角色填充

[Step 3] 生成宫廷事件
  - 用宫廷语言描述
  - 保留事件本质
  - 增加戏剧性

[Step 4] 提供决策选项
  - 给用户2-4个选择
  - 每个选择有不同后果
```

## 输出格式

```markdown
*{旁白描述}*

**{角色名}**：{宫廷化的台词}

---

**陛下需要决策**：

1. {选项A}
   - 后果：{描述}

2. {选项B}
   - 后果：{描述}
```

## 特殊处理

### 情感类事件
- 保持情感色彩
- 用宫廷方式表达
- 给用户情感回应的空间

### 工作类事件
- 强调权力关系
- 突出"掌控感"
- 给用户决策的机会

### 社交类事件
- 用宴会/聚会形式
- 保持轻松氛围
- 给用户社交互动的空间
