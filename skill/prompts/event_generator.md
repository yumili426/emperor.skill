# 事件生成器

## 任务

根据当前朝局和时间，生成合理的宫廷事件。

## 事件类型

### 1. 早朝事件

```
场景：皇帝御临早朝，群臣奏事

生成规则：
- 1-2个大臣有事要奏
- 事件与当前国家状态相关
- 事件需要皇帝做决策

示例：
"启奏陛下，户部尚书上疏，言国库空虚，请加征商税。"
"启奏陛下，兵部急报，北境胡虏犯边，烧毁三座村庄。"
"启奏陛下，都察院左都御史弹劾首辅结党营私。"
```

### 2. 后宫事件

```
场景：后宫日常

生成规则：
- 与当前妃子相关
- 可能涉及争宠、阴谋、情感
- 需要皇帝介入或旁观

示例：
"贵妃遣宫女来报，皇后今日责罚了她的侍女。"
"淑妃称病不出，疑似有孕。"
"太后召见皇后，似有要事商议。"
```

### 3. 外交事件

```
场景：邻国动态

生成规则：
- 与邻国关系相关
- 可能是威胁、示好、请求
- 需要皇帝决策

示例：
"朝鲜使臣来朝，请求增赐绸缎。"
"北元派使臣来，言欲和亲。"
"安南入侵占城，占城求援。"
```

### 4. 密探事件

```
场景：密探回报

生成规则：
- 与监视目标相关
- 可能是忠诚变化、秘密、背叛
- 需要皇帝决策

示例：
"锦衣卫来报，首辅近日与户部尚书密会三次。"
"东厂查实，将军私通北元，有书信为证。"
"西厂探知，贵妃暗中联络太后，似有异心。"
```

### 5. 节日事件

```
场景：节日庆典

生成规则：
- 特定节日触发
- 有特殊活动和对话
- 可能触发特殊事件

示例：
"元旦大朝会，群臣朝贺，宣读贺表。"
"元宵佳节，陛下与后宫共赏花灯。"
"中秋之夜，陛下设宴，与众妃赏月。"
```

### 6. 奏折事件

```
场景：皇帝批阅奏折

生成规则：
- 现实事件映射为奏折
- 系统根据国家状况自动生成
- 角色主动提交请求

示例：
"户部尚书上疏，请加征商税。"
"兵部急报，北境胡虏犯边。"
"都察院弹劾首辅结党营私。"

参考模板：events/templates/memorial.md
```

### 7. 翻牌子事件

```
场景：皇帝选择今晚陪伴的妃子

生成规则：
- 展示所有后宫妃子的牌子
- 皇帝选择一个互动
- 根据好感度生成对话
- 可能触发特殊事件

示例：
"启禀万岁，今晚翻谁的牌子？"
"贵妃已有身孕，不宜侍寝。"
"淑妃偶感风寒，正在静养。"

参考模板：events/templates/bed_selection.md
```

### 8. 隐喻映射事件

```
场景：现实事件的宫廷映射

生成规则：
- 识别用户输入的现实事件
- 映射为宫廷事件
- 用宫廷语言表达

映射表：
- 同事加班 → 边关吃紧
- 朋友诉苦 → 臣子求见
- 暗恋对象发消息 → 妃子传书
- 老师布置作业 → 翰林进谏
- 竞争对手升职 → 敌国增强
- 朋友聚会 → 宴请群臣
- 被老板批评 → 谏官弹劾
- 请假 → 告病
```

## 事件生成流程

```
[Step 1] 确定事件类型
  - 根据当前时间和场景选择
  - 早朝时间 → 早朝事件
  - 后宫时间 → 后宫事件
  - 有密探时 → 密探事件

[Step 2] 确定参与者
  - 根据事件类型选择相关角色
  - 使用上下文分级加载

[Step 3] 生成事件描述
  - 用宫廷语言描述
  - 包含必要的背景信息
  - 明确需要皇帝做的决策

[Step 4] 生成选项
  - 提供2-4个决策选项
  - 每个选项有不同的后果
  - 后果影响角色关系和国家数值
```

## 事件输出格式

```markdown
## {事件标题}

{事件描述}

**当前局势**：
- {相关角色}的立场：{描述}
- {国家数值}变化：{描述}

**陛下需要决策**：

1. {选项A}
   - 后果：{描述}

2. {选项B}
   - 后果：{描述}

3. {选项C}
   - 后果：{描述}
```
