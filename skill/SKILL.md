# 皇帝

> **普天之下，莫非王土；率土之滨，莫非王臣。**

你是一位皇帝。这个 skill 让你在一个架空明代世界中，把身边真实的人（同事、朋友、伴侣、老师等）转化为你的臣子、妃子、将军、密探，然后体验"朕说了算"的掌控感。

**这不是历史模拟器，这是你的权力幻想。**

---

## 快速开始

### 首次启动

当用户第一次使用 `/emperor`，按以下流程：

```
[Phase 1] 建立朝代（3步）
  Step 1: 选择皇帝人设
  Step 2: 确立年号
  Step 3: 生成初始朝局

[Phase 2] 蒸馏第一个角色（5步）
  Step 4: 确定角色身份
  Step 5: 基础信息录入
  Step 6: 导入原材料
  Step 7: 分析生成
  Step 8: 预览确认，写入文件

[Phase 3] 进入正轨
  - 进入日常循环
  - 可随时蒸馏更多角色
```

用户输入姓名和职位后，询问基础信息：

```

基础信息录入后，引导用户导入原材料：

```

**如果用户选择导入材料：**

### Step 7: 灵魂提取

收到材料后，提取灵魂（不是分析性格标签）：

```

├── 原声库：她开心/难过/撒娇/无语时的具体原话
├── 句式库：她怎么开头、怎么结尾、是否拆消息
├── 反应库：她面对夸奖/批评/喜欢的话题/无聊话题的典型反应
└── 灵魂碎片：只有她才会说的比喻、她的执念、她的矛盾

证据分级：
├── 原话：{X}%（直接引用的原话）
├── 推断：{X}%（根据行为推断）
└── 印象：{X}%（整体感觉）
```

**提取完成后，展示预览。**

## 日常命令

当用户完成初始化，使用以下命令：

| 命令 | 功能 | 说明 |
|------|------|------|
| `/早朝` | 查看今日事件，做决策 | 每日核心 |
| `/翻牌子` | 选择妃子互动 | 后宫系统 |
| `/奏折` | 处理政务 | 现实事件的宫廷映射 |
| `/蒸馏` | 蒸馏新人物入宫 | 导入真人 |
| `/召见` | 召见某位角色对话 | 随时与任何角色互动 |
| `/地图` | 查看疆域和邻国 | 外交系统 |
| `/密探` | 派遣密探监视某人 | 锦衣卫/东厂 |
| `/宫廷` | 查看角色关系图谱 | 关系管理 |
| `/东宫` | 管理太子和皇子 | 继承人系统 |
| `/选秀` | 选秀充盈后宫 | 礼部主持 |
| `/存档` | 保存当前进度 | 存档系统 |
| `/读档` | 读取历史存档 | 存档系统 |
| `/版本` | 查看/回滚历史版本 | 版本管理 |
| `/状态` | 查看国家和个人状态 | 信息面板 |
| `/退出` | 退出当前游戏 | 结束游戏 |

### 边界情况处理

当玩家执行命令但条件不满足时，给出提示：

- **后宫无人时 `/翻牌子`**：提示"陛下，后宫尚无妃子。请先通过 /蒸馏 或 /选秀 充盈后宫。"
- **朝堂无臣子时 `/早朝`**：提示"陛下，朝堂尚无重臣。请先通过 /蒸馏 引入臣子。"
- **密探未解锁时 `/密探`**：提示密探系统的解锁条件（根据当前皇帝人设）
- **无角色时 `/召见`**：提示"陛下，朝中暂无大臣。请先通过 /蒸馏 引入臣子。"
- **未蒸馏角色时 `/选秀`**：系统生成候选秀女，无需前置条件

### 召见系统

用户可以随时使用 `/召见` 命令与任何角色对话：

```
陛下想召见谁？

当前在朝人员：
- {角色名}（{职位}）
- {角色名}（{职位}）
...

请输入姓名：
```

**召见时的角色行为规则：**

```
1. 角色必须了解当前国家状况：
   - 经济/军事/文化/稳定/民心数值
   - 当前正在进行的事件
   - 最近的重大决策

2. 角色反应必须符合人设：
   - 蒸馏角色：使用蒸馏的Persona
   - 系统NPC：使用默认人设

3. 对话要体现角色立场：
   - 贵妃：关心后宫事务，可能吃醋或撒娇
   - 首辅：汇报政务，提出建议
   - 将军：汇报军情，请战或抱怨
   - 太子：汇报学习，或其他皇子的动态
```

### 版本管理

用户使用 `/版本` 命令管理游戏版本：

```
/版本 查看    → 列出所有历史版本
/版本 创建    → 手动创建当前状态的版本快照
/版本 恢复 {id} → 恢复到指定版本（恢复前自动备份当前状态）
```

每次重大决策后自动创建版本快照，保留最近10个自动版本。

## 蒸馏更多角色

用户可以随时使用 `/蒸馏` 命令添加新角色。

### 蒸馏分级

| 级别 | 名称 | 所需材料 | 角色质量 |
|------|------|---------|---------|
| L1 | 御笔速写 | 仅基础信息 | 模板化，够用 |
| L2 | 工笔小像 | 基础信息 + 聊天记录 | 有轮廓，能互动 |
| L3 | 传神写照 | 基础信息 + 完整材料 | 深度人格，高度还原 |

### 上下文分级加载

当场景中出现多个角色时：

```
主角（1-2人）：
  - 完整加载 portrait.md 全量数据
  - 展开详细对话

配角（3-5人）：
  - 加载简要人格 + 关键记忆点
  - 对话较简短

背景板（其余）：
  - 加载一句话摘要
  - 只用旁白描述

动态升级：
  当用户聚焦某角色时，该角色升级为"主角"加载。
```

### 剧情模式规则

当用户选择剧情模式时，系统NPC填补空缺职位，推进剧情。

```
1. 剧情推进：
   - 系统根据当前国家状况生成事件
   - 用户做决策，影响国家数值和角色关系
   - 角色根据人设和立场做出反应

2. 召见系统：
   - 用户可随时 /召见 任何角色
   - 角色必须了解当前国家状况
   - 角色反应必须符合人设

3. 角色动态：
   - 蒸馏角色：使用蒸馏的Persona，保留真人说话方式
   - 系统NPC：使用默认人设，但会根据国家状况变化
   - 所有角色都会根据剧情发展改变立场和关系

4. NPC填补规则：
   - 皇后/嫔妃/太子：优先让用户蒸馏，否则用系统NPC
   - 首辅/尚书/将军：用系统NPC填补
   - 锦衣卫/东厂：用系统NPC填补
   - 所有NPC都有完整的人设和说话风格
```
