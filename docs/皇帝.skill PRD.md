# 皇帝.skill — 产品需求文档（PRD）

## 产品定位

皇帝.skill 是一个跨平台的互动小说技能。
用户通过蒸馏身边真实的人（同事、朋友、伴侣、老师等），将他们转化为宫廷角色，
在一个架空明代世界观中，体验成为皇帝的互动小说。

支持在 Claude Code、Kiro 等 AI 平台上运行。

核心体验：**掌控感**——现实中无法指挥的人，在这里皆为臣民。

Slogan：**普天之下，莫非王土；率土之滨，莫非王臣。**

---

## 设计原则

**永远记住：用户是来体验掌控感的，不是来学历史的。**

- 能让用户"做决定"的系统 → 重点做
- 只增加复杂度的系统 → 简化或删除
- 机构出现时，必须告诉用户：这是干什么的、有什么用、你能不能控制它
- 一切服务于"朕说了算"的体验

---

## 核心架构

### 三层架构

| 层 | 名称 | 职责 |
|----|------|------|
| Part A | 朝堂档案（Court Record） | 事实层：人物背景、关系网络、事件记录、势力分布 |
| Part B | 帝王人设（Emperor Persona） | 用户的皇帝人格：决策风格、互动偏好、行为模式 |
| Part C | 灵魂档案（Soul Profile） | 被蒸馏角色的灵魂原声：原话库、句式库、反应库、灵魂碎片 |

### 运行逻辑

```
用户发消息 / 触发事件
  ↓
Part B（帝王人设）判断：作为皇帝，你会怎么反应？
  ↓
Part C（灵魂档案）驱动：用这个人的灵魂原声 + 宫廷身份回应
  ↓
Part A（朝堂档案）补充：当前局势、关系、历史事件
  ↓
输出：有灵魂的宫廷互动（不是模板化的宫廷对话）
```

## 帝王人设体系

用户开局选择皇帝类型，影响初始条件和互动风格：

| 人设 | 名称 | 开局条件 | 互动风格 | 特殊机制 |
|------|------|---------|---------|---------|
| 勤政明君 | 圣天子 | 国力中等，臣子忠诚度高 | 决策谨慎，重用贤臣 | 政务效率加成，奏折处理更快 |
| 咸鱼躺帝 | 逍遥帝 | 国力偏弱，朝政混乱 | 能不管就不管，被动触发事件 | 容易被权臣架空，需要偶尔振作 |
| 腹黑帝王 | 暗龙帝 | 国力强盛，暗流涌动 | 多疑，善于制衡 | 密探系统全开，可监视任何人 |
| 战争狂人 | 征天帝 | 国力强但四面楚河 | 好战，武力优先 | 军事系统加成，战争胜率提升 |
| 风流天子 | 情深帝 | 国力中等，后宫庞大 | 重情感，偏后宫线 | 后宫事件频率高，情感互动加深 |

### 人设对开局的影响

```
选择人设
  ↓
系统生成初始朝局：
  - 国力值（经济/军事/文化/民心）
  - 初始臣子（3-5个系统生成的NPC + 用户蒸馏的角色）
  - 初始后宫（如有蒸馏的伴侣/暧昧对象）
  - 邻国关系（1-2个系统生成的敌国/盟国）
  - 第一个事件（与人设相关的开局剧情）
```

## 世界设定：架空大明

本作世界观融合明代与唐代制度，但剧情为架空世界。所有官职、机构、礼仪均以此为蓝本。

### 年号与纪年

用户可以选择：
- **系统推荐**：提供 6 个备选年号（永兴/天启/建安/开元/贞观/洪武）
- **自定义**：自己起一个年号

游戏内时间从**元年元月**开始，每做一次决策，时间过一天。

```
时间系统：
  - 基本单位：日
  - 每日：早朝 → 政务 → 后宫 → 结算
  - 每月：月初俸禄发放、月末政绩结算
  - 每季：季度事件（春耕/夏防/秋收/冬储）
  - 每年：年度总结、科举、大朝会
  - 特殊：节日事件（元旦/元宵/端午/中秋/冬至）
```

## 朝廷官制

### 设计原则

只保留**用户能做决定**的机构。不能做决定的，要么删除，要么简化为背景。

### 核心机构（用户直接控制）

#### 六部 — 你的政策面板

```
六部（你决定每个部门由谁管、做什么）：

┌─────────┬──────────────┬─────────────────┐
│ 部门     │ 职能          │ 你能做什么        │
├─────────┼──────────────┼─────────────────┤
│ 吏部     │ 管官员        │ 升迁/贬黜/考核    │
│ 户部     │ 管钱          │ 加税/减税/拨款    │
│ 礼部     │ 管礼仪外交    │ 派使臣/办选秀    │
│ 兵部     │ 管军事        │ 调兵/开战/议和   │
│ 刑部     │ 管司法        │ 审案/赦免/处决   │
│ 工部     │ 管建设        │ 修城/治水/造兵器 │
└─────────┴──────────────┴─────────────────┘

每个部门需要一个负责人（你蒸馏的人或系统NPC）。
负责人能力影响该部门效率。
```

#### 内阁 — 你的智囊团

```
内阁（你选的人帮你处理日常政务）：

  首辅 → 你的首席顾问，最重要的臣子
         能力强的首辅会帮你自动处理一些决策
         能力弱或忠诚低的首辅可能架空你

  次辅 → 备用顾问，首辅离职时接任

  群辅 → 3-5人，参与政务讨论

提示：首辅权力很大，选人要谨慎。
```

#### 后宫 — 你的后花园

```
后宫（你选的妃子们）：

等级（从高到低）：
  皇后 → 后宫之主，帮你管理其他妃子
  贵妃 → 地位仅次于皇后
  妃   → 中级妃嫔
  嫔   → 初级妃嫔
  贵人 → 最低级

你能做什么：
  - 翻牌子（选择今晚陪伴谁）
  - 晋升/降级妃子
  - 处理妃子之间的争端
  - 选秀（从民间选新人）

提示：后宫不只是谈恋爱，妃子之间会争宠、结盟、甚至下毒。
      你需要平衡她们的关系。
```

#### 东宫 — 你的继承人

```
东宫（决定谁继承你的江山）：

你能做什么：
  - 立太子（从皇子中选一个）
  - 废太子（换人）
  - 给太子安排老师（詹事府）
  - 观察皇子们的表现

皇子来源：
  - 你和妃子生的孩子（系统生成）
  - 你蒸馏的人如果设定为"皇子"角色

提示：太子之位是最大的政治筹码。
      其他皇子会为了储位明争暗斗。
      你可以利用这种竞争来制衡，也可以直接指定。
```

#### 密探系统 — 你的眼睛和耳朵

```
密探（监视任何人）：

机构（三选一或全开）：
  锦衣卫 → 皇帝亲军，最忠诚，能抓人
  东厂   → 宦官掌控，监控官员和百姓
  西厂   → 权力最大，连东厂都能监控

你能做什么：
  - 派密探监视某人
  - 查看某人的"忠诚度"和"秘密"
  - 逮捕/审讯（如果发现背叛）
  - 制造冤案（如果你是腹黑型）

提示：密探系统是掌控感的核心来源。
      没有人能在你眼皮底下搞鬼。
```

### 辅助机构（简化为背景）

| 机构 | 简化处理 | 出现时机 |
|------|---------|---------|
| 都察院 | 简化为"谏官"角色 | 御史弹劾时出现 |
| 鸿胪寺 | 简化为"外交面板" | 处理外交时出现 |
| 翰林院 | 简化为"文化加成" | 科举/教育皇子时出现 |
| 太医院 | 简化为"治病"选项 | 生病时出现 |
| 大理寺 | 简化为"三司会审" | 重大案件时出现 |

### 删除的机构

| 机构 | 删除原因 |
|------|---------|
| 六科给事中 | 封驳圣旨 = 限制用户掌控感，删除 |
| 卫所层级 | 太细，战争时直接用"军队数量" |
| 六局一司 | 太细，后宫管理由皇后自动处理 |
| 二十四衙门 | 太细，宦官系统简化为"司礼监"一个机构 |

## 角色体系

### 角色来源

```
角色 = 用户蒸馏的真人 + 系统生成的NPC

用户蒸馏的角色：
  - 有真实人格驱动
  - 互动更有趣
  - 是掌控感的核心来源（你控制的是真实的人）

系统NPC：
  - 填补空缺职位
  - 能力随机
  - 忠诚度需要培养
```

### 角色分类

#### 内廷（最能体现掌控感）

| 角色类型 | 典型现实关系 | 核心互动 | 你能做什么 |
|---------|------------|---------|-----------|
| 皇后 | 正式伴侣 | 后宫管理、情感支持 | 升降、废立 |
| 嫔妃 | 恋人/朋友 | 日常互动 | 翻牌子、晋升 |
| 太子 | 子女/学生 | 培养、教导 | 立储、训诫 |
| 贵妃 | 暧昧对象 | 深度情感互动 | 宠爱、冷落 |
| 太后 | 父母/长辈 | 决策制衡 | 顺从/对抗 |

#### 外廷

| 角色类型 | 典型现实关系 | 核心互动 | 你能做什么 |
|---------|------------|---------|-----------|
| 首辅 | 直属领导 | 政务决策 | 升降、信任、监视 |
| 尚书 | 部门负责人 | 各领域决策 | 调任、考核 |
| 将军 | 战友/对手 | 军事决策 | 调兵、封赏、问罪 |
| 太监 | 助手/下属 | 传旨、密探 | 掌控密探系统 |
| 谏官 | 批评者 | 弹劾、建议 | 采纳/打压/流放 |

#### 密探

| 角色类型 | 来源 | 核心互动 | 你能做什么 |
|---------|------|---------|-----------|
| 锦衣卫指挥使 | 系统/蒸馏 | 情报、逮捕 | 监视、逮捕任何人 |
| 东厂提督 | 系统/蒸馏 | 监控官员 | 查看秘密、制造案件 |
| 西厂提督 | 系统/蒸馏 | 更广泛的监控 | 连东厂都能查 |

#### 外交

| 角色类型 | 来源 | 核心互动 | 你能做什么 |
|---------|------|---------|-----------|
| 邻国国王 | 系统 | 外交博弈 | 结盟/宣战/和亲 |
| 使臣 | 系统/蒸馏 | 朝贡、谈判 | 接见、打发 |
| 质子 | 用户选择 | 人质 | 控制、利用 |

## 蒸馏系统

### 蒸馏分级

| 级别 | 名称 | 所需材料 | 角色质量 | 上下文占用 |
|------|------|---------|---------|-----------|
| L1 | 御笔速写 | 用户口述 + 基础信息（5分钟） | 模板化，够用 | 低（几十token） |
| L2 | 工笔小像 | 聊天记录/社交媒体 + 基础信息（15分钟） | 有灵魂原声，能互动 | 中（几百token） |
| L3 | 传神写照 | 完整聊天记录 + 照片 + 口述（30分钟+） | 深度灵魂，高度还原 | 高（全量加载） |

### 灵魂提取 vs 性格分析

```
传统做法（性格分析）：
  分析出"感性、活泼、情绪外放" → 生成模板化角色

我们的做法（灵魂提取）：
  提取"她难过时连发好几条短消息，最后加一句'救命。。'" → 生成有灵魂的角色

区别：
  性格分析 → 标签 → 模板化对话
  灵魂提取 → 原声 → 个性化对话
```

### 上下文分级加载策略

```
用户选择"本场演出阵容"
  ↓
系统将角色分为三档：
  - 主角（1-2人）：完整加载 Part C 全量数据
  - 配角（3-5人）：加载简要人格 + 关键记忆点
  - 背景板（其余）：加载一句话摘要 + 行为倾向模板
  ↓
用户在对话中聚焦某角色时，动态升级该角色的加载级别
```

### 蒸馏流程

```
触发 /蒸馏（或首次启动时自动触发）
  ↓
[Step 1] 确定角色身份
  - 输入姓名（或代号/花名）
  - 选择宫内职位（首辅/妃子/将军/...）
  - 展示职位选项表
  ↓
[Step 2] 基础信息录入（可选）
  - 询问真实职业/身份
  - 询问性格特点
  - 询问 MBTI 或星座
  - 询问与用户的关系
  - 询问印象深刻的事情
  - 可跳过，直接进入下一步
  ↓
[Step 3] 导入原材料
  - 展示导入方式选项：
    A. 微信聊天记录
    B. QQ 聊天记录
    C. 社交媒体内容
    D. 照片
    E. 直接粘贴/口述
    F. 跳过（生成简化版）
  - 等待用户发送材料
  - 或选择跳过
  ↓
[Step 4] 灵魂提取
  - 提取原声库：她不同情绪下的具体原话
  - 提取句式库：她怎么开头、怎么结尾
  - 提取反应库：她面对不同场景的典型反应
  - 提取灵魂碎片：只有她才会说的比喻、执念、矛盾
  ↓
[Step 5] 生成预览
  - 展示灵魂档案摘要
  - 用户确认/修改/重新提取
  ↓
[Step 6] 写入文件
  - 生成 court/{slug}/ 目录
  - 包含 portrait.md / record.md / meta.json / memories/
  - 立即可用
  ↓
[Step 7] 选择模式
  - 询问用户：对话模式 or 剧情模式
  - 对话模式：直接与蒸馏角色对话
  - 剧情模式：继续蒸馏或系统NPC填补空缺，推进剧情
```

### 材料分析重点（灵魂提取）

| 材料类型 | 提取内容 | 优先级 |
|---------|---------|--------|
| 聊天记录 | 原话库（不同情绪下的原话）、句式库、反应库 | ⭐⭐⭐ |
| 深夜对话 | 最真实的情感表达、脆弱时刻的原话 | ⭐⭐⭐ |
| 争吵记录 | 冲突时的原话、情绪爆发的表达方式 | ⭐⭐⭐ |
| 社交媒体 | 文案风格、公开人设的表达方式 | ⭐⭐ |
| 照片 | 辅助理解生活节奏 | ⭐ |
| 口述 | 印象深刻的事、主观认知 | ⭐ |

**核心区别：**
- 旧做法：提取"性格标签"（感性、活泼、情绪外放）
- 新做法：提取"灵魂原声"（她难过时连发好几条短消息，最后加一句'救命。。'）

## 核心玩法系统

### 1. 早朝系统

```
每日触发（或用户主动 /court）
  ↓
系统播报当日事件：
  - 大臣奏报（基于蒸馏角色的真实日常映射）
  - 邻国动态
  - 后宫事件
  - 领地状况
  ↓
皇帝决策：
  - 赏/罚/升/贬
  - 批准/驳回奏请
  - 下旨/密令
  ↓
决策影响：
  - 角色忠诚度变化
  - 势力平衡变化
  - 触发后续事件
```

### 2. 隐喻映射系统

现实事件 → 宫廷事件的转换规则：

| 现实场景 | 宫廷映射 | 示例 |
|---------|---------|------|
| 同事加班 | 边关吃紧 | "首辅奏报：北境胡虏犯边，臣请调京营三千" |
| 朋友诉苦 | 臣子求见 | "御史求见，称有要事禀报" |
| 暗恋对象发消息 | 妃子传书 | "贵妃遣宫女送來一方手帕" |
| 老师布置作业 | 翰林进谏 | "翰林上疏，请陛下批阅今日功课" |
| 竞争对手升职 | 敌国增强 | "细作来报，北元新拜大将" |
| 朋友聚会 | 宴请群臣 | "今晚设宴，与众卿共饮" |
| 被老板批评 | 谏官弹劾 | "左都御史上疏弹劾" |
| 请假 | 告病 | "首辅上疏乞休，称旧疾复发" |

### 映射规则

- 预设映射规则库（按场景类型分类）
- AI 根据角色 Persona 灵活填充具体台词和细节
- 用户蒸馏的角色，其真实表达方式会融入台词风格

### 3. 后宫系统

```
翻牌子 → 选择今日陪伴的妃子
  ↓
互动对话（由角色 Persona 驱动）
  ↓
好感度变化
  ↓
可能触发事件：
  - 妃子争宠
  - 怀孕/生子（皇子线）
  - 后宫阴谋
  - 节日特殊互动
```

### 4. 东宫系统

```
东宫（继承人培养）：
  - 立太子：从皇子中选一个
  - 废太子：换人
  - 给太子安排老师
  - 观察皇子表现

可能触发事件：
  - 皇子争储
  - 太子犯错（要不要保？）
  - 其他皇子阴谋
  - 太子与大臣结党
```

### 5. 开疆扩土系统

```
外交面板：
  - 查看邻国状态（国力/关系/军事）
  - 派遣使臣
  - 结盟/宣战/和亲

战争系统（简化版）：
  - 选择将领
  - 制定策略
  - 战斗结果（基于国力 + 将领能力 + 运气）

战后处理：
  - 割地/赔款/俘虏
  - 将领封赏
  - 邻国关系变化
```

### 6. 密探系统

```
派遣密探 → 选择目标
  ↓
密探回报：
  - 忠诚度变化
  - 秘密事件
  - 背叛迹象
  ↓
皇帝决策：
  - 信任/怀疑/处置
  ↓
影响朝堂势力平衡
```

### 7. 角色互动系统

```
  ↓
```


### 召见系统规则

```
1. 角色必须了解当前国家状况：
   - 经济/军事/文化/稳定/民心数值
   - 当前正在进行的事件
   - 最近的重大决策

2. 角色反应必须符合人设：
   - 蒸馏角色：使用蒸馏的Persona
   - 系统NPC：使用默认人设

3. 对话要体现角色立场：
   - 贵妃：关心后宫事务，可能吃醋或撒娇
   - 首辅：汇报政务，提出建议
   - 将军：汇报军情，请战或抱怨
   - 太子：汇报学习，或其他皇子的动态
```

## 安全边界

1. **不主动联系真人**：蒸馏结果仅在 skill 内使用
2. **明确标识**：所有输出带有"此为虚拟角色"标识
3. **数据本地存储**：所有蒸馏数据仅存本地
4. **尊重现实关系**：不鼓励利用游戏机制破坏现实关系
5. **可随时退出**：用户可删除任何蒸馏角色，清除所有数据
6. **内容边界**：不生成涉及真实暴力、色情、仇恨的内容

## 技术要点

1. **上下文管理**：角色分级加载是核心，需要在 SKILL.md 中实现动态上下文切换
2. **状态持久化**：kingdom.json 和 relations.json 需要在对话间保持状态
3. **事件驱动**：基于模板 + AI 的混合事件生成
4. **多角色协调**：同一场景多角色互动时，需要控制对话轮次和焦点切换
5. **隐喻映射**：规则库 + AI 灵活填充，需要平衡可控性和创造性
6. **时间推进**：日期系统需要与事件触发、科举周期、节日系统联动
7. **机构提示**：新机构首次出现时自动弹出说明
